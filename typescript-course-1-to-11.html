Lesson 1 — What is TypeScript?

What is TypeScript?

TypeScript is a programming language created by Microsoft in 2012. It is basically JavaScript with types.

Types help developers catch errors before running the program, making code more reliable and easier to maintain.

Example

let age: number = 25;

age = 30; // ✅ OK

// age = "hello" ❌ error
// Cannot assign string to number

🎯 Why Use TypeScript?

✅ Catches Errors Early

Find bugs while coding, not in production

🚀 Better Code Completion

Your editor knows what properties exist

📚 Self-Documenting Code

Types explain what data is expected

🛡️ Safer Refactoring

Change code confidently without breaking things

  • 1. Node.js

    Download from nodejs.org

  • 2. Install TypeScript
    npm install -g typescript

📝 Your First TypeScript File

Create hello.ts:

function greet(name: string): string {

}

console.log(greet("World")); // Hello, World!

Compile: tsc hello.ts   Run: node hello.js

hello.ts
→ tsc →
hello.js

TypeScript compiles to JavaScript. Type checking happens during compilation!

Lesson 2 — Primitive Types

In TypeScript, primitives are the most basic types: string, number, boolean, bigint, symbol, null, and undefined.

📝 string

let name: string = "Ruth";
let greeting = `Hello, ${name}!`;

🔢 number

let age: number = 20;
let price: number = 9.99;
let hex: number = 0xf00d;

✅ boolean

let isStudent: boolean = true;

🔟 bigint

let big: bigint = 12345678901234n;

🔑 symbol

let id: symbol = Symbol("id");
// Always unique

⚠️ null / undefined

let empty: null = null;

Lesson 3 — Arrays & Tuples

📚 Arrays

Method 1: Type[]

let numbers: number[] = [1, 2, 3];
let names: string[] = ["Alice", "Bob"];

Method 2: Array<Type>

let numbers: Array<number> = [1, 2, 3];

🎯 Tuples

Fixed length + specific types at each position.

let person: [string, number] = ["Alice", 25];
let color: [number, number, number] = [255, 0, 128];

🔧 Common Array Methods

let fruits: string[] = ["apple", "banana", "orange"];

fruits.push("grape");   // add to end
fruits.pop();           // remove from end
fruits.length;          // 3

let upper = fruits.map(f => f.toUpperCase());
let long  = fruits.filter(f => f.length > 5);

Lesson 5 — Functions

🎯 Typing Functions

function add(a: number, b: number): number {
  return a + b;
}

const multiply = (a: number, b: number): number => a * b;

function log(message: string): void {
  console.log(message);
}

⚙️ Optional & Default Parameters

function greet(name: string, title?: string): string {
  return title ? `Hello, ${title} ${name}` : `Hello, ${name}`;
}

function createUser(name: string, role: string = "guest"): void {
  console.log(`${name} is a ${role}`);
}

📝 Rest Parameters

function sum(...numbers: number[]): number {
  return numbers.reduce((total, n) => total + n, 0);
}

sum(1, 2, 3);        // 6
sum(10, 20, 30, 40); // 100

Lesson 7 — Classes

NEW

Classes are blueprints for creating objects. TypeScript adds access modifiers and strong typing on top of JavaScript classes, making Object-Oriented Programming (OOP) safer and more expressive.

🏗️ Basic Class

class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age  = age;
  }

  greet(): string {
    return `Hi, I'm ${this.name} and I'm ${this.age} years old.`;
  }
}

const alice = new Person("Alice", 25);
console.log(alice.greet()); // Hi, I'm Alice and I'm 25 years old.

🔐 Access Modifiers

TypeScript provides three access modifiers to control visibility:

public

Accessible from anywhere (default)

private

Only accessible inside the class

protected

Accessible inside the class and subclasses

class BankAccount {
  public owner: string;
  private balance: number;

  constructor(owner: string, initialBalance: number) {
    this.owner   = owner;
    this.balance = initialBalance;
  }

  deposit(amount: number): void {
    this.balance += amount;
  }

  getBalance(): number {
    return this.balance; // ✅ can access private inside class
  }
}

const account = new BankAccount("Alice", 1000);
account.deposit(500);
console.log(account.getBalance()); // 1500
// account.balance ❌ Error: balance is private

⚡ Shorthand Constructor

TypeScript lets you declare and assign properties directly in the constructor parameters: 

// Instead of this (verbose):
class Dog {
  name: string;
  breed: string;
  constructor(name: string, breed: string) {
    this.name  = name;
    this.breed = breed;
  }
}

// Write this (shorthand):
class Dog {
  constructor(
    public name: string,
    public breed: string
  ) {}
}

const rex = new Dog("Rex", "Labrador");
console.log(rex.name); // Rex

🧬 Inheritance

Classes can extend other classes to inherit their properties and methods: 

class Animal {
  constructor(public name: string) {}

  speak(): string {
    return `${this.name} makes a sound.`;
  }
}

class Dog extends Animal {
  constructor(name: string, public breed: string) {
    super(name); // call parent constructor
  }

  speak(): string {
    return `${this.name} barks!`; // override parent method
  }
}

const dog = new Dog("Rex", "Labrador");
console.log(dog.speak()); // Rex barks!

🔄 Getters & Setters

class Temperature {
  private _celsius: number = 0;

  get fahrenheit(): number {
    return this._celsius * 9/5 + 32;
  }

  set celsius(value: number) {
    if (value < -273.15) {
      throw new Error("Temperature below absolute zero!");
    }
    this._celsius = value;
  }
}

const temp = new Temperature();
temp.celsius = 100;
console.log(temp.fahrenheit); // 212

🎭 Abstract Classes

Abstract classes are templates — they can't be instantiated directly. Subclasses must implement abstract methods: 

abstract class Shape {
  abstract getArea(): number; // must be implemented

  describe(): string {
    return `This shape has area: ${this.getArea()}`;
  }
}

class Circle extends Shape {
  constructor(private radius: number) { super(); }

  getArea(): number {
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(private w: number, private h: number) { super(); }

  getArea(): number { return this.w * this.h; }
}

const c = new Circle(5);
console.log(c.describe()); // This shape has area: 78.53...

Lesson 8 — Generics

NEW

Generics let you write reusable, type-safe code that works with different data types. Instead of writing the same function multiple times for each type, you write it once and let TypeScript figure out the type.

🧬 The Problem Generics Solve

// Without generics — we'd need a separate function per type:
function getFirstNumber(arr: number[]): number { return arr[0]; }
function getFirstString(arr: string[]): string { return arr[0]; }

// With generics — ONE function handles ANY type:
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

getFirst<number>([1, 2, 3]);      // returns 1 (number)
getFirst<string>(["a", "b"]);    // returns "a" (string)
getFirst([true, false]);           // TypeScript infers boolean!

📦 Generic Interfaces

interface ApiResponse<T> {
  data: T;
  success: boolean;
  message: string;
}

interface User {
  name: string;
  email: string;
}

// Response containing a User
const userResponse: ApiResponse<User> = {
  data: { name: "Alice", email: "alice@example.com" },
  success: true,
  message: "User found"
};

// Response containing a number
const countResponse: ApiResponse<number> = {
  data: 42,
  success: true,
  message: "Count retrieved"
};

🔒 Generic Constraints

Use extends to limit what types are allowed:

// T must have a .length property
function logLength<T extends { length: number }>(value: T): void {
  console.log(value.length);
}

logLength("hello");       // 5 ✅ (strings have .length)
logLength([1, 2, 3]);     // 3 ✅ (arrays have .length)
// logLength(42);         // ❌ numbers don't have .length

// Multiple type parameters
function merge<T, U>(obj1: T, obj2: U): T & U {
  return { ...obj1, ...obj2 } as T & U;
}

const merged = merge({ name: "Alice" }, { age: 25 });
// merged is { name: string, age: number }

Lesson 9 — Enums

NEW

Enums (short for enumerations) let you define a set of named constants. They make your code more readable by replacing magic numbers or strings with meaningful names.

🔢 Numeric Enums

enum Direction {
  Up,    // 0
  Down,  // 1
  Left,  // 2
  Right  // 3
}

let move: Direction = Direction.Up;
console.log(move); // 0

// Custom starting value
enum Priority {
  Low    = 1,
  Medium = 2,
  High   = 3
}

let taskPriority: Priority = Priority.High;
console.log(taskPriority); // 3

📝 String Enums

String enums are more readable and preferred in most modern TypeScript code:

enum Status {
  Pending  = "PENDING",
  Active   = "ACTIVE",
  Inactive = "INACTIVE"
}

enum UserRole {
  Admin  = "admin",
  User   = "user",
  Guest  = "guest"
}

function checkRole(role: UserRole): void {
  if (role === UserRole.Admin) {
    console.log("Welcome, admin!");
  }
}

checkRole(UserRole.Admin); // Welcome, admin!

⚡ const Enums (Optimized)

Adding const before enum tells TypeScript to inline the values at compile time for better performance: 

const enum Color {
  Red   = "RED",
  Green = "GREEN",
  Blue  = "BLUE"
}

let favoriteColor: Color = Color.Blue;
// Compiles to: let favoriteColor = "BLUE"
// The enum itself is removed from the output!

Lesson 10 — Type Narrowing & Type Guards

NEW

Type narrowing is how TypeScript figures out a more specific type from a broader one. This is essential when working with union types or values that could be different things.

🔍 typeof Guard

function formatValue(value: string | number): string {
  if (typeof value === "string") {
    // TypeScript knows value is a string here
    return value.toUpperCase();
  } else {
    // TypeScript knows value is a number here
    return value.toFixed(2);
  }
}

console.log(formatValue("hello")); // "HELLO"
console.log(formatValue(3.14159)); // "3.14"

🏛️ instanceof Guard

class Cat {
  meow() { return "Meow!"; }
}

class Dog {
  bark() { return "Woof!"; }
}

function makeSound(animal: Cat | Dog): string {
  if (animal instanceof Cat) {
    return animal.meow(); // TypeScript knows it's a Cat
  }
  return animal.bark();   // TypeScript knows it's a Dog
}

makeSound(new Cat()); // "Meow!"
makeSound(new Dog()); // "Woof!"

🗝️ "in" Operator Guard

interface Admin {
  name: string;
  adminLevel: number;
}

interface RegularUser {
  name: string;
  email: string;
}

function describeUser(user: Admin | RegularUser): string {
  if ("adminLevel" in user) {
    return `Admin level: ${user.adminLevel}`; // Admin
  }
  return `User email: ${user.email}`; // RegularUser
}

🔖 Type Assertions (as)

Sometimes you know more than TypeScript. Use as to tell TypeScript the exact type: 

// When fetching from DOM
const input = document.getElementById("username") as HTMLInputElement;
console.log(input.value); // TypeScript now knows .value exists

// When you know the data shape
const data: unknown = { name: "Alice", age: 25 };
const user = data as { name: string; age: number };
console.log(user.name); // Alice

// ⚠️ Use with caution — you take responsibility for the type!

⚠️ Warning: Type assertions bypass TypeScript's checks. Only use them when you're certain about the type.

Lesson 11 — Modules & Organizing Code

NEW

As your projects grow, you'll split code into multiple files (modules). TypeScript fully supports ES Modules — the modern way to organize and share code between files.

📤 Exporting

// math.ts

// Named exports
export function add(a: number, b: number): number {
  return a + b;
}

export function subtract(a: number, b: number): number {
  return a - b;
}

export const PI: number = 3.14159;

// Default export (one per file)
export default function multiply(a: number, b: number): number {
  return a * b;
}

📥 Importing

// app.ts

// Import named exports
import { add, subtract, PI } from "./math";

// Import default export
import multiply from "./math";

// Import everything as a namespace
import * as Math from "./math";

console.log(add(2, 3));       // 5
console.log(subtract(10, 4)); // 6
console.log(PI);              // 3.14159
console.log(multiply(3, 4));  // 12
console.log(Math.add(1, 2));  // 3

🏷️ Exporting Types & Interfaces

// types.ts
export interface User {
  id: number;
  name: string;
  email: string;
}

export type Status = "active" | "inactive" | "banned";

// userService.ts
import type { User, Status } from "./types"; // import type (preferred)

function getUser(id: number): User {
  return { id, name: "Alice", email: "alice@example.com" };
}

📁 Recommended Project Structure

my-project/
├── src/
│   ├── types/
│   │   └── index.ts       ← all shared types & interfaces
│   ├── utils/
│   │   └── helpers.ts     ← utility functions
│   ├── services/
│   │   └── userService.ts ← business logic
│   └── index.ts           ← main entry point
├── tsconfig.json           ← TypeScript configuration
└── package.json

💡 tsconfig.json controls how TypeScript compiles your project. Run tsc --init to generate it automatically.
